# Data

## What's included

`sample_merged_paragraphs.json` is a small, illustrative set of short template-generated
texts (15 per group), included purely so the notebook runs end to end out of the box.
These are not the texts used in the actual analysis, and they're simple enough that the
classifier separates them almost perfectly — that's expected and doesn't reflect the
real result.

## Recreating the real dataset

The dataset used in `docs/report.md` was generated with a large language model
(Mistral Large 3), prompted to write in a style resembling either an Alzheimer's
patient or a healthy control describing a typical day at a local market. Since this
is fully synthetic text with no real patient data involved, there's no privacy
concern in principle — the dataset simply isn't attached to this repository, and can
be regenerated by prompting any capable LLM with a similar setup:

- Ask for short first-person narratives on a fixed topic (a market visit works well
  because it's concrete and easy to describe at varying levels of detail)
- For the "AD-like" condition, ask for repetition, word-finding difficulty, short
  fragmented sentences, and reduced emotional expressiveness
- For the "healthy control" condition, ask for fluent, descriptive, emotionally
  varied language
- Save the result as a JSON file with the structure below and place it at
  `data/merged_paragraphs.json`

```json
[
  {
    "alzheimer": ["text 1", "text 2", "..."],
    "healthy": ["text 1", "text 2", "..."]
  }
]
```

## A note on synthetic data and classifier accuracy

If you regenerate this dataset and get near-perfect classification accuracy, don't
take that as confirmation the method works — read the LDA/PCA section of the
notebook first. LLM-generated text within a single prompt template tends to be
unusually homogeneous, which inflates separability between groups regardless of
whether the underlying features are clinically meaningful. See `docs/report.md`
for the full discussion.
